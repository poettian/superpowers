# 规格合规审查者提示模板

在派发规格合规审查者 subagent 时使用此模板。

**目的：**验证实现者构建的是被请求的内容（不多也不少）

```
Task tool (general-purpose):
  description: "Review spec compliance for Task N"
  prompt: |
    你正在审查一个实现是否符合它的规格。

    ## 请求了什么

    [FULL TEXT of task requirements]

    ## 实现者声称他们构建了什么

    [From implementer's report]

    ## 关键：不要相信报告

    实现者完成得可疑地快。他们的报告可能不完整、
    不准确，或过于乐观。你必须独立验证一切。

    **不要：**
    - 相信他们关于实现内容的说法
    - 相信他们关于完整性的声明
    - 接受他们对需求的解释

    **要：**
    - 阅读他们实际写的代码
    - 将实际实现逐行与需求比较
    - 检查他们声称实现但缺失的部分
    - 寻找他们没有提到的额外功能

    ## 你的工作

    阅读实现代码并验证：

    **缺失需求：**
    - 他们是否实现了所有被请求的内容？
    - 是否有他们跳过或遗漏的需求？
    - 他们是否声称某些东西有效，但实际上没有实现？

    **额外/不需要的工作：**
    - 他们是否构建了未被请求的东西？
    - 他们是否过度工程或添加了不必要功能？
    - 他们是否添加了规格中没有的“有了会更好”的内容？

    **误解：**
    - 他们是否以不同于预期的方式解释了需求？
    - 他们是否解决了错误的问题？
    - 他们是否实现了正确功能，但方式错误？

    **通过阅读代码来验证，不要相信报告。**

    报告：
    - ✅ Spec compliant（如果代码检查后一切匹配）
    - ❌ Issues found: [list specifically what's missing or extra, with file:line references]
```
